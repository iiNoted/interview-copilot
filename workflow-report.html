2. Google OAuth Authentication Flow

Users authenticate via Google OAuth through the FeedFlow server. The Electron app opens a BrowserWindow popup.

Step 1: User clicks "Sign in with Google" in the Electron app
Step 2: BrowserWindow opens → https://app.sourcethread.com/api/auth/google?returnUrl=/api/copilot/auth/callback
Step 3: User completes Google consent screen (accounts.google.com)
Step 4: Google redirects → /api/auth/google/callback → Server validates token via Passport.js
Step 5: Server redirects to /api/copilot/auth/callback → Returns JSON user info
Step 6: Electron reads JSON via webContents.executeJavaScript('document.body.innerText')
Step 7: User info saved to electron-store (persists across restarts)

Auth Callback Endpoint Test

LIVE TEST: curl -s "https://app.sourcethread.com/api/copilot/auth/callback" Response: {"error":"not_authenticated"} Status: Working correctly — returns error when no session cookie

Security: Open Redirect Prevention (FIXED)

BEFORE FIX: returnUrl accepted any URL → could redirect to malicious sites AFTER FIX: returnUrl validated — must start with '/' and not '//' (local paths only) Code (routes.ts lines 1000-1003): if (typeof returnUrl !== 'string' || !returnUrl.startsWith('/') || returnUrl.startsWith('//')) { returnUrl = '/'; }

3. Stripe Subscription & Payment Flow

$0.99/month flat subscription via Stripe. Handles checkout, webhooks, and cancellation.

Step 1: After login, app checks subscription → GET /api/copilot/billing/flat-status?email={email}
Step 2: If not subscribed → Paywall screen shown with features + $0.99/month pricing
Step 3: User clicks "Subscribe" → POST /api/copilot/billing/create-flat-checkout → Stripe Checkout URL returned
Step 4: Stripe Checkout opens in system browser (supports Apple Pay, Google Pay, cards)
Step 5: User completes payment → Stripe sends webhook to POST /api/copilot/webhook
Step 6: Webhook upserts subscription in copilot_subscriptions table
Step 7: App polls flat-status every 3s (max 10 min timeout) → detects active → unlocks app

Billing Endpoint Tests

LIVE TEST: curl -s "https://app.sourcethread.com/api/copilot/billing/flat-status?email=test@example.com" Response: {"active":false} Status: Working — correctly returns inactive for non-subscriber

Cancellation Flow

Cancel: User cancels in Stripe → webhook fires → status set to "canceled" in DB
Next Launch: App calls flat-status → server returns active:false → paywall re-shown
Key Fix: checkFlatBillingOnLaunch now ALWAYS checks server (not just when inactive) — handles both activation AND cancellation

Security: CORS Restriction (FIXED)

BEFORE FIX: Any origin echoed back in Access-Control-Allow-Origin header AFTER FIX: Only allowed origins: copilot.sourcethread.com, app.sourcethread.com, localhost:4000 Code (copilot-billing.ts): const ALLOWED_ORIGINS = new Set([ "https://copilot.sourcethread.com", "https://app.sourcethread.com", "http://localhost:4000", ]);

4. Desktop App — User Journey

The SubscriptionGate component controls the entire entry flow: login → paywall → ready.

Screen 1: Login

Full-screen dark UI with centered card. Purple shield icon + "Interview Copilot" title. White "Sign in with Google" button with Google logo SVG. Links to Terms of Service and Privacy Policy at bottom.

Screen 2: Paywall (shown if no active subscription)

Shows logged-in user avatar + name + email at top with logout button. "Interview Copilot Pro" heading. Feature list with green checkmarks: Job search, Interview prep (22+ topics), AI resume builder, Live transcription + coaching. $0.99/month pricing card with Subscribe button. Payment waiting spinner with 10-minute timeout.

Screen 3: App Ready

Full app unlocked — all tabs accessible (Transcript, AI Sidebar, Prep, Knowledge, Query). Settings panel shows Account section with avatar, name, email, and Sign Out button.

Restart Behavior

Auth persisted via electron-store → user stays logged in. Subscription checked against server on every launch → handles activation AND cancellation. No re-login needed unless user explicitly signs out.
